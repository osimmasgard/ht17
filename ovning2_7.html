Övning 2.7

Webbsidelayouter (fixerad, flytande och elastisk)

Hur man programmatiskt väljer att designa sin hemsida kommer att ha en påverkan på användarens upplevelse av webbsidan. Det är flera faktorer som en webbutvecklare måste ta hänsyn till, exempelvis:

  • Besökarens skärmupplösning
  • Vilken webbläsare som används
  • Om fönstret är maximerat, har verktygsfält öppna etc
  • Vilken hårdvara besökaren använder
Vilken webbsidelayout man väljer kommer att få konsekvenser för de saker jag listade så det kan vara viktigt att välja rätt. Det första stora valet är huruvida man vill använda CSS eller tabeller för att skapa layouten, då det går att använda båda. CSS anses av de flesta vara det bästa sättet av främst dessa anledningar:
  • Webbsidan blir lättare att arbeta med eftersom man lyfter ut designen från innhållet i en css-fil
  • Tabeller kräver många fler taggar för att bygga upp sidan, det blir fort rörigt! Exempelvis <table>, <tr> och <td>
  • Eftersom det finns beroenden mellan rader och kolumner så blir det lätt krångligt att uppdatera en komplex webbsida utan att ha sönder den relationen
  • Webbläsare måste lägga ner mer tid för att rendera en sida som är uppbyggd med hjälp av tabeller då den måste läsa igenom sidan ett par gånger innan den kan ta vissa renderingsbeslut.
Här kommer en kort genomgång av de metoder man kan välja mellan om man använder CSS som layoutmetod:

En fixerad layout använder en fast ram runt elementen som bygger upp sidan. Elementen innanför ramen styr man sedan storleken på genom att använda storleksmåttet %. På så vis kommer användare av olika skärmstorlekar/upplösning alltid att få rätt proportioner på de element som finns på sidan. Det svåra här är att välja rätt storlek på den fasta ramen.

En flytande layout använder ingen fast ram runt elementen, utan de använder sig enbart av procentenheten för styrning av elementens bredd.

Den sista layouttypen, elastisk, tar sig an problemet med hjälp av "em". Måttenheten em, som beskrevs i övning 2.2 är ett dynamiskt attribut som har en relation till en användarens förvalda fontstorlek. Denna storlek är olika på skärmar med olika upplösning, vilket gör att en "em" är väldigt användbart som måttenhet. Det som talar emot denna layouttyp är att den är mer tidskrävande att utveckla med och det kan vara svårare att testa sig fram till en bra konfiguration för just den webbsida man utvecklar.

Några viktiga attribut

Position

h2 { position: absolute;
left: 100px;
top: 150px;
}

Position kan vara static,absolute, fixed, relative, sticky, initial eller inherit där static är defaultvärdet. Static betyder att element ritas ut i den ordning som de är skrivna i html-dokumentet. Absolute betyder att elementet är placerat i absolut relation med föräldraelementet. Fixed betyder att elementet är placerat i relation till webbläsarfönstret. Relative betyder att elementet är placerat i relation till den position det skulle ha normalt sett. Sticky betyder att positionen är relativ till hur långt besökaren scrollat på webbsidan. Initial sätter tillbaka värdet till grundvärdet. Slutligen: inherit, bestämmer att elementet ärver egenskapen "position" från sin förälder.

  • top: styr avståndet till nästa element ovanför
  • left: styr avståndet till elementet till vänster
  • bottom: styr avståndet till elementet under detta
  • right: styr avståndet till elementet till höger
  • z-index: styr hur element ska synas om de ligger ovanför varandra. Ett högre värde gör att den relativt de andra elementen hamnar högre upp i stapeln och därmed ligger ovanpå ett element med ett lägre z-index
  • float: används ofta för att få element att flyta vid sidan av andra element, exempelvis en bild vid sidan av en text
  • clear: används för att styra om element ska tillåtas "floatas" vid sidan av detta element eller inte. Man kan tillåta att detta görs på vissa sidor enbart.

Storlek

  • width: styr bredden på elementet
  • height: styr höjden på elementet
  • max-width: styr elementets maximala bredd
  • max-height: styr elementest maximala höjd
  • min-width: styr elementets minimala bredd
  • min-height: styr elementets minimala höjd
  • vertical-align: styr hur elementet ska placeras i höjdled i det element som det befinner sig i. Exempelvis kan man få text att ligga mot "taket" av raden istället för mot botten.
