Övning 2.1

Externt länkade CSS-filer

CSS används för att styra formattering och utseende på en websida. Har man en stor websida där man önskar göra en mängd undantag från default-reglerna (mer om dessa senare) så är det bättre att lyfta ut CSS-reglerna till en extern fil som man sedan länkar in i sin webbsida. Skulle reglerna var specificerade i samma fil så skulle dokumentet vara svårt att läsa och överblicka, och ändringar skulle inte göras lika lätt. En annan fördel med att lägga reglerna i en extern fil är att man då enkelt kan länka in dem i flera dokument, och på så vis centraliserat styra utseendet på sin webbplats.

Arv

Inom programmeringen är "arv" en vanligt förekommande företeelse. Anledningen till detta är man ofta bygger upp strukturer och att det uppstår relationer mellan elementen i dessa, man talar om "förälder" och "barn" (eng: parent/child), eller "branch" och "leaf". En förälder ligger hierariskt på en högre nivå än ett barn och precis som i verkligheten så kan ett barn ärva egenskaper från sin förälder även inom programmeringens värld. Inom CSS så används denna regel i högsta grad, så placerar man element inom andra element så kommer vissa egenskaper ärvas ner från det yttre elementet. De vanligaste egenskaperna som omfattas av arv är de som har med textutseende etc att göra, andra egenskaper som inte lämpar sig lika bra att ärva ner omfattas inte av arvregeln.

Cascade

Cascade syftar på i vilken ordning CSS-regler tillämpas och vilken som slutligen gäller. Det finns 3 grundnivåer i vilken CSS kan defineras. Dels så kommer varje webläsare med sin egen CSS-"regelbok" som beskriver hur alla taggar ska visas och bete sig, denna kallas för "User Agent Stylesheet" ("User Agent" betyder fritt översatt "webläsarversion och tillverkare"). Som ett andra steg så kan varje användare ange att en CSS-fil ska tillämpas mot varje websida som besöks (User Stylesheet). På så vis kan man styra att man exempelvis inte vill ha en vit bakgrundsfärg på alla websidor, utan att den ska vara rosa. Den sista nivå som CSS-sätts på är genom den CSS som webutvecklaren har specificerat (Author Stylesheets), antingen direkt i html-koden eller inlänkat från en css-fil. I slaget om vilken css-specifikation som ska gälla för ett visst element så vinner den specifikation som har högst "tyngd". Denna tyngd bestäms utifrån ett regelverk där User Agent Stylesheet har lägst tyngd och Author Stylesheet har högst vikt. Man kan frångå denna ordning i sin User Stylesheet genom lägga till "!important" till den regel ska vara tyngst och på så vis få det som man vill oavsett vad Author Stylesheet har satt för regel.

Selektorer

Selektorerna berättar för webläsaren som tolkar CSS:en var regeln ska sättas. Exempelvis kan varje html-tagg vara en selektor i sig, dvs om vi i CSS-filen skriver att för <body> ska regeln "background-color: grey" sättas, så är body i det här fallen en selektor.

Det finns olika typer av selektorer:

  1. Tagg-selektorer används för att styra element med en viss tagg, exempelvis taggen <body>.
  2. Attribut-selektorn pekar på element som har ett visst attribut satt, exempelvis om ett visst element har attributet "target" satt såsom här: <a target="_blank">
    3. så kommer attribut-selektorn kunna styra alla element med "target" satt. Det går även att styra så att attribut med ett visst värde enbart, exempelvis 'target="_blank"'
  3. ID-selektorer är sådan som pekar på element där ett visst id har satts. Exempelvis: <img src="lib/images/fot.png" id="bild">
  4. Klass-selektorer är sådan som pekar på element där en viss klass har satts. Exempelvis: <img src="lib/images/fot.png" class="bild">
  5. Barn-selektorer pekar på element som är barn av ett annat element, exempelvis ul li { margin: 0 0 5px 0; } som pekar på "list-items" som är barn till "unordered lists". Denna relation måste således finnas för att css-stylen ska appliceras.

Enheter

Historien varför CSS har flera måttenheter finns att hitta i tryckkonstens historia. Man har därför fler måttenheter än vad som borde behövas för att tillfredsställa olika typer av användare.
  • in: står för "inches", dvs måttenheten tum.
  • cm: står för centimeter.
  • mm: står för måttenheten millimeter.
  • pt: står för måttenheten "point".
  • pc: står för måttenheten "pica".
  • px: står för "pixel" och syftar till datornskärmens pixlar. Denna måttenhet gör att det ser olika ut beroende på vilken pixelmängd man har på skärmen eftersom avstånden mäts i antalet pixlar.
  • em: är en måttenheten som används för typsnitt, fonter. Den är relativ och inte absolut som de ovan, vilket betyder att den kommer se olika ut på olika skärmar, men med målsättningen att se relativt sett bra ut oberoende av skärm, dvs den krymper och blir större i relation med skärmen.
  • ex: denna relativa måttenhet används sällan. Den är knuten till ett typsnitts "x-height" som är olika för olika fonter, dock kan man säga att den är ungefär höjden på gemenerna a, c, m eller o.
  • %: syftar till procenten av storleken av det omgivande elementet, eller procenten av normalstorleken av elementet. Exempelvis: p {font-size: 16pt; line-height: 125%;}

Här är skillnaden mellan de absoluta måttenheterna (förutom px): 1in = 2.54cm = 25.4mm = 72pt = 6pc

Hur ett CSS-dokument är uppbyggt

I ett CSS-dokument så listar man upp sina selektorer och hur de ska deklareras (styras), det är inte mer komplicerat än så! Exempelvis: selektor { attribut1: värde1 ; attribut2: värde2}
